Model Introduction


In brief, Association rule mining uses a combination of if else statements that help discover relationships within the data, doing so by looking for patterns, correlations, and associations between the several variables in a dataset.

The association rule has 2 parts:

• Antecedent (if)
• Consequent (then)

The consequent is usually found in combination with the antecedent. For illustration purposes, assume we have a dataset of all Netflix shows watched by all USA Netflix users, the following association rules are derived from the data:

• If a user watches "Mr Robot", there is a 65% chance they watch "Dark".
• If a user watches "The Witcher", there is a 85% chance they watch "Stranger Things".

The logic is applicable many problems like consumer clustering, recommendation systems, etc. where there are correlational and/or causal relationships between several variables within the data.

As for how ARM was used in this project, i decided to use it on music lyrics to properly understand if there are any patterns and relationships between words in popular music. The dataset used has a combination of all the billboards top 100 songs from the years 1956 till 2022, which encompasses a wide variety of music genres, styles, and trends.

How it works



Just like any machine other model that is built on text data, the first step is to pre process the text data and clean it to make it appropriate for cleaning. The processing steps include the following:

• Removing stopwords ("the" , "a", "an", "and", etc) that would create unnecessary noise in the model.
• Replacing \n with spaces
• Lemmitizing words, which means bringing words to their base form Example: Caring --> Care, Plays --> Play, etc.
• Run a sentiment analysis on every sentence.

After pre processing, the data was ready for apriori modelling, which is an algorithm that uses frequent itemsets to generate association rules, and it is designed to work on datasets that contain transactions. With the help of these association rules, it determines how strongly or how weakly two objects are connected. The algorithm's parameters are the following:

• min_support: Used to select the items with support values greater than the value specified
• min_confidence: Filters those rules that have confidence greater than the confidence threshold specified by the parameter.
• min_lift: Specifies the minimum lift value for the short listed rules.
• min_length : Specifies the minimum number of items that you want in your rules.

Playing around with these parameters has a huge impact on the final results. For example, increasing the min support parameter would lead to a “stricter” model and therefore have less associations, which is something i did since myy dataset had over 4000 songs, which would have led to a hectic and messy association results.
